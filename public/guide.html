    Overview

    Product Analyzer is a competitive intelligence and AI-powered marketing platform. It helps you collect competitor data, analyze advertising strategies, and generate marketing assets — all organized inside containers.

    The platform follows a four-phase workflow: Collect data, Analyze it with AI, Propose a strategy via Magic AI Proposal, then Generate marketing assets informed by the proposal and competitive insights.

    Quick Start

    1. Create a Container — Click "New Container" and enter your product name and URL. A container holds all data for one product or project.
    2. Add Your Product Details — Fill in the "My Product" section with your product's description, features, and target audience.
    3. Add Competitors — Enter competitor names and their Facebook Ad Library / Google Ads Transparency URLs.
    4. Collect — Run the Ad Scraper to collect competitor ads. Connect Google Ads to pull your own campaign data. Add metadata, case studies, and notes.
    5. Analyze — Run the Scraped Ads Analyzer, SEO Analyzer, and GAds Analysis. These produce AI insights from your collected data.
    6. Push to Context — Push your best analyses to the Container Context collector. This is the knowledge base that all downstream agents read.
    7. Propose — Run the Magic AI Proposal. It reads all context + raw data to produce a comprehensive marketing strategy with creative briefs.
    8. Generate — Use generation tools (Landing Page, Quiz, Image Ads, etc.) to create assets. Some read the proposal directly, all read Container Context.

    Containers

    A container is the central organizing unit. It holds everything related to one product or project: your product info, competitors, scraped ads, analyses, generated assets, and curated context.

    Every tool in the app operates within a container. You can have multiple containers for different products or campaigns. Each container stores its data in an independent JSON file.

    My Product Your product name, URL, description, features, and audience
    Competitors Competitor names and Ad Library / Transparency URLs
    Entries (Metadata) Notes, HTML snippets, links, user feedback, A/B test results
    Settings FB Pixel, GA4 tag, custom code injection, user context notes

    Data Collection

    Scrapes live competitor ads from Facebook Ad Library and Google Ads Transparency Center. It uses a headless browser to navigate the pages, captures screenshots of each ad, and extracts ad copy, images, and metadata.

    To use: add a competitor with their Ad Library or Transparency URL, then click "Scrape". Results appear as cards showing each ad's creative, copy, and link. You can also validate scrape quality with the Scrape Validator.

    Connects to your own Google Ads account via API to pull campaign performance data: clicks, impressions, CTR, conversions, cost, ad copy, and keywords. This is your own ad data — not competitor data.

    You can then run AI analysis on your campaigns to identify what's working and what isn't. The analysis results get pushed to Container Context as "Google Ads Performance" insights, which generation agents (Proposal, Keyword Strategist, Test Ideator, Container Chat) read automatically to produce better-informed output.

    Manually add information to your container: user feedback, product specs, A/B test results, strategy notes, HTML snippets, or any reference material. These entries become part of the container's knowledge base.

    Paste a URL or text of a marketing case study and the AI will extract key lessons: what worked, target audience, channels used, metrics achieved, and actionable takeaways relevant to your product.

    Analysis

    AI-powered analysis of a competitor's scraped ads. It examines ad copy patterns, messaging angles, CTAs, visual strategies, and targeting approaches. The output includes: key themes, content patterns, strengths, weaknesses, and opportunities for your product.

    Run this after scraping a competitor to understand their advertising strategy. Each analysis can be pushed to Container Context.

    Analyzes websites and pages directly — not ads. Two modes: Competitor SEO studies a competitor's website to extract strategies, keyword patterns, and content structures you can learn from. Own Product SEO audits your product's site with scores and improvement recommendations.

    Results include keyword gaps, content opportunities, technical SEO issues, and competitive positioning insights. Push results to Container Context for use in generation.

    The Container Context is a curated insights collector. When you push an analysis, SEO report, or Google Ads data to context, it gets summarized into a natural-language brief and stored.

    Generation agents (Proposal, Landing Page, etc.) automatically read all context items to inform their output. The more relevant context you push, the better the generated results.

    Use "Push All" to push all available data at once, or push individual sections from their analysis pages.

    Propose

    The central strategy step. The Proposal reads all container data — competitor analyses, SEO insights, Google Ads performance, scraped ads, metadata, and all Container Context items — to generate a comprehensive marketing strategy.

    The output includes positioning strategy, messaging frameworks, creative briefs, channel priorities, budget allocation, and a timeline. This is the most data-hungry tool — push as much context as possible before running it.

    Downstream agents use the proposal: Prompt Generator requires a completed proposal, while Image Ad Creator and RPS Test Ideator read it for messaging context. Run the proposal before generating assets for best results.

    Generation

    Generates a keyword strategy with categories, search intent mapping, content recommendations, and priority rankings. Uses container context to tailor keywords to your market position and competitive landscape.

    Creates a complete, ready-to-deploy HTML landing page with responsive design, persuasive copy, and conversion-focused layout. Automatically injects your Facebook Pixel and GA4 tracking codes if configured in Settings.

    Creates an interactive lead-generation quiz as a standalone HTML page. The quiz guides prospects through questions that segment them, then presents personalized product recommendations. Includes tracking code injection.

    Generates image ad concepts with detailed specifications: dimensions, visual layout, copy placement, color schemes, and design notes. Provides ready-to-brief specs for designers or AI image tools. Reads the latest proposal for messaging context if one exists.

    Requires a completed proposal. Takes the proposal's creative briefs and generates detailed, ready-to-use prompts for AI image tools (Midjourney, DALL-E, Stable Diffusion). Each prompt is tailored to the strategy and messaging outlined in the proposal.

    Generates A/B test plans organized around the KNOWNS/UNKNOWNS framework. Identifies what you know about your audience and market (knowns) and what assumptions need testing (unknowns), then designs specific experiments to validate them. Reads the latest proposal and Google Ads data for richer context.

    Brainstorms new product ideas, features, and positioning angles based on your scrape data and competitor analyses. Useful for finding market gaps and differentiation opportunities.

    AG-015 Container Chat

    Container Chat is now accessible from the top navigation bar on all pages via the "Chat" link.

    Container Chat lets you ask free-form questions about your container data. The AI reads your container context and answers questions like:

    • "What messaging patterns do my competitors share?"
    • "Which competitor has the strongest value proposition?"
    • "Suggest three headline angles based on my competitor analysis"
    • "Summarize the key takeaways from all my analyses"

    The more context you've pushed, the more informed the chat responses will be.

    Tips & Best Practices

    Recommended workflow order: Scrape Ads → Analyze Ads → SEO Analysis → Push to Context → Run Proposal → Generate Assets (Landing Pages, Quizzes, Image Ads, Prompts, etc.)
    Push context early and often. Every generation tool reads container context. The more insights you push, the more tailored and accurate the outputs become.
    Use Container Chat to explore. Before generating a full proposal, ask the chat questions to test hypotheses and explore angles. It's faster than running a full generation.
    Add metadata manually. Don't rely only on scraped data. Add your own customer feedback, A/B test results, brand guidelines, and product specs. This grounds the AI in real-world data that scraping can't capture.
    Configure Settings. Add your Facebook Pixel ID and GA4 Measurement ID in container Settings so generated landing pages and quizzes automatically include tracking. You can also add custom context notes that all agents will see.
    Run multiple competitors. Analysis quality improves with more data points. Scrape and analyze at least 3 competitors before generating proposals for the best results.
