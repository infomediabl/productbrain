    Overview

    ProductBrain is a competitive intelligence and AI-powered marketing platform. It helps you collect competitor data, analyze advertising strategies, and generate marketing assets — all organized inside containers.

    The platform follows a four-phase workflow: Collect data, Analyze it with AI, Propose a strategy via Magic AI Proposal, then Generate marketing assets informed by the proposal and competitive insights.

    Quick Start

    1. Create a Container — Click "New Container" and enter your product name and URL. A container holds all data for one product or project.
    2. Add Your Product Details — Fill in the "My Product" section with your product's description, features, and target audience.
    3. Add Competitors — Enter competitor names and their Facebook Ad Library / Google Ads Transparency URLs.
    4. Collect — Run the Ad Scraper to collect competitor ads. Connect Google Ads to pull your own campaign data. Add metadata, case studies, and notes.
    5. Analyze — Run the Scraped Ads Analyzer, SEO Analyzer, and GAds Analysis. These produce AI insights from your collected data.
    6. Push to Context — Push your best analyses to the Container Context collector. This is the knowledge base that all downstream agents read.
    7. Propose — Run the Magic AI Proposal. It reads all context + raw data to produce a comprehensive marketing strategy with creative briefs.
    8. Generate — Use generation tools (Landing Page, Quiz, Image Ads, etc.) to create assets. Some read the proposal directly, all read Container Context.

    Containers

    A container is the central organizing unit. It holds everything related to one product or project: your product info, competitors, scraped ads, analyses, generated assets, and curated context.

    Every tool in the app operates within a container. You can have multiple containers for different products or campaigns. Each container stores its data in an independent JSON file.

    My Product Your product name, URL, description, features, and audience
    Competitors Competitor names and Ad Library / Transparency URLs
    Entries (Metadata) Notes, HTML snippets, links, user feedback, A/B test results
    Settings FB Pixel, GA4 tag, custom code injection, user context notes

    Data Collection

    Scrapes live competitor ads from Facebook Ad Library and Google Ads Transparency Center. It uses a headless browser to navigate the pages, captures screenshots of each ad, and extracts ad copy, images, and metadata.

    To use: add a competitor with their Ad Library or Transparency URL, then click "Scrape". Results appear as cards showing each ad's creative, copy, and link. You can also validate scrape quality with the Scrape Validator.

    Each ad card in scrape details has two action buttons:

    • Push — pushes the ad's headline, description, CTA, and image path to Container Context for use by downstream agents.
    • Clone — opens a modal to generate a new ad image via OpenRouter AI, adapted for your product. Choose model, format (1:1, 9:16, 16:9), and add custom instructions. Cloned images are saved to the clonedAd/ folder.

    The Container Context is a curated insights collector. When you push an analysis, SEO report, or Google Ads data to context, it gets summarized into a natural-language brief and stored.

    Generation agents (Proposal, Landing Page, etc.) automatically read all context items to inform their output. The more relevant context you push, the better the generated results.

    Use "Push All" to push all available data at once, or push individual sections from their analysis pages.

    Propose

    The central strategy step. The Proposal reads all container data — competitor analyses, SEO insights, Google Ads performance, scraped ads, metadata, and all Container Context items — to generate a comprehensive marketing strategy.

    The output includes positioning strategy, messaging frameworks, creative briefs, channel priorities, budget allocation, and a timeline. This is the most data-hungry tool — push as much context as possible before running it.

    Downstream agents use the proposal: Prompt Generator requires a completed proposal, while Image Ad Curator and RPS Test Ideator read it for messaging context. Run the proposal before generating assets for best results.

    Clones your top-performing Facebook ads into live Taboola native advertising campaigns via the Taboola Backstage API. Uses a 2-step preview flow: AI generates Taboola-optimized copy first, you review and edit the rewrites side-by-side against the originals, then confirm to launch the live campaign.

    Setup (one-time)

    1. Go to Taboola Backstage → Account Settings → API to get your OAuth Client ID, Client Secret, and Advertiser Account ID.
    2. In your container, click Settings → scroll to Taboola Integration.
    3. Enter all three credentials. Click Test Connection to verify they work.
    4. Click Save Settings. Credentials are saved per-container, so different containers can use different Taboola accounts.

    How to use

    1. Scrape first — You need FB ads in your container. Run the Ad Scraper (AG-001) on at least one competitor.
    2. Push context — Push competitor analyses and product info to Container Context. The AI reads this when rewriting ad copy for Taboola's native format.
    3. Open the section — On the container dashboard, find "Taboola Campaign Cloner" (AG-018) and click to expand.
    4. Select ads — Check the FB ads you want to clone. Use "Select Top Ads" to auto-pick proven (non-new) ads. Each ad shows its thumbnail and headline.
    5. Configure — Set campaign name (auto-suggested), daily budget (default $20), CPC bid (default $0.50), country targeting (default US), and platforms (Desktop, Phone, Tablet).
    6. Step 1: Preview Copy — Click "Preview Copy". The AI generates Taboola-optimized titles (<60 chars, curiosity/teaser style) and descriptions (<150 chars). No campaign is created yet.
    7. Step 2: Review & Launch — Click "Review & Launch" to open the preview report. You'll see a side-by-side comparison of each original FB ad vs the AI-rewritten Taboola version. You can edit the title and description directly before confirming.
    8. Confirm — Click "Confirm & Launch on Taboola" to create the live campaign. The system creates the campaign, uploads images, and creates all ad items with your approved copy.
    9. Monitor — When done, click "View Report" to see campaign details, or "Open in Taboola" to go to the Taboola Ads Manager.

    What you get

    The campaign report shows: side-by-side comparison of original FB ads vs Taboola rewrites, Taboola campaign ID, campaign URL, each cloned item with its status, budget and targeting summary, and any errors for individual items.

    Tip: Start with a low daily cap ($10–$20) and 1–3 proven ads. The preview step lets you tweak every headline before going live — use it to add product-specific angles the AI might have missed.

    Analyzes your container data — product info, competitor analyses, scrape results, SEO data, and curated context — to propose spin-off product and business ideas adjacent to your current market.     Folder Ad Importer

    Import ad creatives from a local folder instead of scraping them from the web. Useful when you have downloaded competitor ads manually or received them from a client.

    Setup:

    • Create a folder at data/uploads/<container-id>/
    • Place image files (.jpg, .png, .webp, .gif) in the folder
    • Optionally add an ads.csv file with columns: filename, title, description, cta, url, competitor_name

    Usage: Click "Check Folder" to verify the folder exists and see file counts. Click "Import Ads" to create a scrape result from the folder contents. Imported scrapes appear in the Ad Scraper history with a "FOLDER" badge.

    Once imported, folder ads can be analyzed by the Scraped Ads Analyzer just like web-scraped ads.

    AG-020 Ad Workshop

    A standalone page with two tabs for creating new ad creatives:

    Tab 1 — Clone Existing Ad: Displays a full-width table of all scraped ads across all scrape results. Each row shows the ad thumbnail, competitor name, source (FB/Google), headline, description, CTA, and a Clone button. Clicking Clone opens a modal where you pick an AI image model, format (1:1, 9:16, 16:9), and optional custom instructions, then generates a new image via OpenRouter.

    Tab 2 — Make New Ad: Uses the Hooks Generator (AG-020) to analyze your scraped ads and generate 8-12 advertising hooks/angles. Each hook includes the angle name, hook text, emotion, target segment, rationale, and visual suggestions. You can clone individual hooks or select multiple hooks and batch-clone them.

    Tip: Run a scrape and competitor analysis first to give the Hooks Generator more data to work with. The more ads in your container, the better the generated hooks.

    AG-022 Content Validator

    A standalone validation page that provides a brutally honest AI audit of your marketing assets against your product's curated context and notes. Supports five content types:

    • Hook — Evaluates audience targeting, pain point accuracy, emotional resonance, scroll-stopping potential
    • Angle — Evaluates strategic fit, differentiation, emotional leverage, scalability across formats
    • Image Ad — Evaluates scroll-stopping power, click likelihood, message clarity, brand alignment
    • Landing Page — Evaluates engagement, retention, CTA clarity, visual flow, conversion likelihood
    • Video Transcript — Evaluates hook strength, retention arc, message clarity, CTA placement

    Each validation returns a verdict (pass / needs_work / fail), a score (1-10), specific strengths and weaknesses, actionable recommendations, and a user perspective note describing how a real customer would perceive the asset.

    The page includes a collapsible Container Context panel (read-only) so you can see what context the AI will use. Add an optional comment to focus the AI on specific aspects.

    After validation, click "View Report" to open a dedicated report page with per-section Push to Context buttons (summary, strengths, weaknesses, recommendations, user perspective) and a Push All button — same pattern as Case Study and SpinOff Ideas reports. The AI prompt is product-first and uses a different evaluation lens per content type (e.g., "You just clicked an ad and landed on this page..." for landing pages).

    Tip: Push competitor analyses, SEO data, and keyword strategies to Container Context before validating — the more context, the more specific and useful the feedback. Use the validation comment to ask about specific concerns (e.g., "Does this hook work for Gen Z?").

    AG-025 Quick Questions

    Quick Questions is a lightweight Q&A box on the container dashboard. Type a question and get a concise AI answer based on your product info, competitors, notes, and curated context.

    Unlike Container Chat (AG-015), Quick Questions is designed for fast, one-shot answers — no conversation history. Each answer includes a confidence level (high/medium/low) and lists which data sources were used.

    AG-017 Web Research

    Web Research is a standalone tool accessible from the top navigation bar via the "Research" link. Unlike other agents, it is not tied to a specific container — it operates globally.

    It works in two phases:

    • Phase 1 — Search: Enter any research topic. The agent uses Claude's built-in web search to find 8-15 high-quality sources (articles, videos, PDFs, social posts).
    • Phase 2 — Summarize: Select which sources to summarize. The agent fetches each page via Puppeteer, then uses Claude to produce a summary, key insights, and relevance notes. A combined brief synthesizes all summaries.

    After summarization, you can push individual summaries or all summaries to any container's Context system using the container dropdown and Push buttons. This feeds your web research directly into the knowledge base that downstream agents (Proposal, Keyword Strategist, etc.) read.

    Available from the dedicated Research page. Past research sessions appear in the sidebar for easy retrieval.

    AG-016 DesireSpring

    DesireSpring is a standalone tool accessible from the top navigation bar. It captures feature ideas for the ProductBrain app itself and generates structured implementation instructions.

    The workflow:

    • Write an idea — Describe a feature you want to add to the app (e.g., "Add Semrush data integration").
    • AI generates instructions — The agent reads the app's CLAUDE.md architecture document and produces step-by-step implementation instructions referencing real files, routes, and patterns.
    • Review & edit — The generated instructions appear in an editable text area. You can modify them before saving.
    • Save to file — Save the final instructions to the instructions/ folder as a text file for future reference.

    Tips & Best Practices

    Recommended workflow order: Scrape Ads → Analyze Ads → SEO Analysis → Push to Context → Run Proposal → Generate Assets (Landing Pages, Quizzes, Image Ads, Prompts, etc.)
    Push context early and often. Every generation tool reads container context. The more insights you push, the more tailored and accurate the outputs become.
    Use Container Chat to explore. Before generating a full proposal, ask the chat questions to test hypotheses and explore angles. It's faster than running a full generation.
    Add metadata manually. Don't rely only on scraped data. Add your own customer feedback, A/B test results, brand guidelines, and product specs. This grounds the AI in real-world data that scraping can't capture.
    Configure Settings. Add your Facebook Pixel ID and GA4 Measurement ID in container Settings so generated landing pages and quizzes automatically include tracking. You can also add custom context notes that all agents will see.
    Run multiple competitors. Analysis quality improves with more data points. Scrape and analyze at least 3 competitors before generating proposals for the best results.
