# Product Analyzer — Refactoring Instructions

> Execute each phase in order. Commit after each phase.

---

## App Context

This is a **Marketing Intelligence Tool** renamed to **Product Analyzer**.

The real-world workflow is sequential:
1. **Input** → Marketing person enters product, competitors, metadata, case studies
2. **Analyze** → Run scrapers, competitor analysis, SEO analysis
3. **Ideate** → Based on steps 1+2, generate strategies and proposals
4. **Generate** → Based on all previous steps, create landing pages, ads, quizzes

## Phase 4: Subagent Architecture

Every tool should be a **subagent** — an independent module with its own context that other parts of the app can call.

Each subagent must define:
- What **inputs** it needs (and from which other agents)
- What **outputs** it produces (for other agents to consume)
- A **run** function

### Critical Rule: Concise Communication

Subagent outputs must be **structured data**, NOT verbose text explanations.

```
❌ BAD:
"Based on our comprehensive analysis of the competitor landscape, we recommend
focusing on the following key areas..."

✅ GOOD:
{
  "top_ads": [{ "id": "ad_123", "platform": "meta", "engagement": "high" }],
  "recommended_angles": ["emotional_trigger", "price_comparison"],
  "keyword_gaps": ["branded_terms", "long_tail_questions"]
}
```

Other subagents need to **understand and use** the data — not read essays. Human-readable formatting is only for the final UI display.

### Convert to subagents:
- Case Study Analyzer
- Scraped Ads Analyzer
- Ad Scraper
- SEO Analyzer
- Keyword Strategy Ideator
- RPS Test Ideator
- Magic AI
- Landing Page Generator
- Quiz Generator
- Image Ad Creator
- **Prompt Generator** — subagent used internally by other agents, but **NOT visible on the page at all**

## Phase 6: Data Flow Between Segments

Wire up so data flows through the workflow:

```
METADATA outputs (product_info, competitor_list, case_study_insights)
    ↓
ANALYZER consumes metadata, outputs (ad_analysis, top_ads, keyword_data, scraped_images)
    ↓
PRODUCT DATA supplements with (campaign_data, keyword_performance)
    ↓
IDEATOR consumes all above, outputs (keyword_strategy, test_variants, full_proposal)
    ↓
GENERATOR consumes all above, produces (landing_pages, quizzes, ad_images)
```

- Use a shared context/store so each subagent can read outputs from previous agents
- Show visual indicators: which agents completed ✅ and which are pending ⏳
- Later-stage agents should indicate what upstream data they're waiting for

---

## Phase 7: Magic AI — Final Proposal

Magic AI is the key output. When it runs (consuming all previous agent data), it must produce:

**Ad Clone Recommendations:**
- Reference the actual competition image (locally saved from scrape)
- Exact prompt to recreate/clone the ad
- Text/copy proposals (headline, body, CTA variants)

**Search Campaign Recommendations:**
- Priority keywords with match types
- Ad group structure
- Budget allocation suggestions

**Angle Analysis:**
- Which competitor angles to clone vs differentiate on
- Ranked by predicted effectiveness

**Concrete Deliverables:**
- Landing pages to create (with briefs)
- Ad images to create (with prompts)
- Tests to run (with hypotheses)

Everything must be **specific and actionable** — referencing real scraped data, real competitor ads, and real keywords. Not generic marketing advice.
